Sincronizar alternativas de texto con la presentación

En ocasiones, cuando el espectador este viendo una película o vídeo no es cómodo tener que ira un enlace externo a leer la transcripción a texto para saber de qué trata dicho recurso. Por ello, es mucho más cómodo para el usuario que la transcripción a texto vaya apareciendo en tiempo real junto con los fotogramas del vídeo.

Poner los enlaces que tienen los mapas de imágenes explícitamente

Usar los colores adecuadamente para enfatizar, así como listas ordenadas

Un orden de preferencia debe ser indicado mediante colores, y para aquellas personas que no los puedan apreciar bien, se usarán listas ordenadas.

  1. Una cosa

  2. Otra cosa

Contraste suficiente

Asegurarse de que el contraste entre el color de la letra y el del fondo sea suficiente para que las personas con dificultades a la hora de difrenecias colores no tengan problemas.

Lenguajes markup antes que imágenes

Si existe un lenguaje markup para expresar información que normalmente expresaríamos mediante una imagen, es imprescindible usarlo, pues el código se podrá entender fácilmente. Por ejemplo, MathML facilita la escritura de expresiones matemáticas.

Usar hojas de estilos

Los atributos de HTML para dar estilo a las fuentes y demás elementos ya quedaron obsoletos, por lo que es una buena práctica utilizar hojas de estilos como las que usará esta página.

Usar medidas relativas más que absolutas

Por ejemplo, en CSS, se debería usar "em" como unidad medida, y no "cm" o "pt" que puede que hagan que el contenido no se redimensione bien en diferentes navegadores.

Utilizar los tamaños de encabezado adecuados para el contenido de la página

Por ejemplo, usar un h1

Usar las listas correctamente

Los elementos de lista OL y UL han de ser usados correctamente (como se ha hecho varias veces en esta página), y no para conseguir sangrías o indentaciones en el texto.

Uso de las etiquetas Q y BLOCKQUOTE

Estas etiquetas han de ser utilizadas únicamente para citas, y no para indentar texto.

Las comillas se ponen solas!

Para indentar texto, podemos usar las propiedades CSS de "margin-left" y derivadas.

Incluir un resumen de la tabla en la etiqueta HTML

Por ejemplo: 

            TABLE border="1" summary="This table charts the number of
            cups of coffee consumed by each senator, 
            the type of coffee (decaf or regular),
            and whether taken with sugar.">
            ... table markup ...

Especificar abreviaciones para las cabeceras

Por ejemplo: 

            TH id="t3" abbr="Type">Type of Coffee/TH

Asegurarse de que el contenido de la página siga siendo legible sin hojas de estilo

Por ejemplo, estableciendo la posición del texto en el propio documento CSS.

            STYLE TYPE="text/css"

                .part1{ color: red; font-size: 14pt;
                padding-left: 0; margin-top: 40px;
                font-family: copperplate gothic bold, fantasy, sans-serif }

De esta forma, estableceremos la posición del texto mediante las propiedades padding, margin ,etc.

Asegurarse de que el contenido sigue siendo accesible con scripts desactivados

Puede ocurrir que algún navegador no soporte scripts, para ello se suele hacer que los scripts no se ejecuten en el cliente, sino en el servidor, además de evitar usar links que utilicen javascript, como por ejemplo "a href="javascript:" "

Scripts indepedendientes del dispositivo de entrada

Es recomendable usar la etiqueta NOSCRIPT tras un script para que los navegadores que no los soporten, puedan mostrar al menos su funcionamiento.

Evitar usar el parpadeo de pantalla

Es bastante importante para la gente que pueda sufrir de epilepsia fotosensible que no se incluyan gifs que puedan causar este tipo de ataques a las personas. No podemos poner un ejemplo pues estaríamos llevándonos la contraria.

Cuando se incluya contenido en movimiento, proporcionar un mecanismo que active y desactive la animacion.

No utilizar la actualizacion automatica de la pagina
